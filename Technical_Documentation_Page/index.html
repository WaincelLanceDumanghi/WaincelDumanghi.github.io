Introduction of CSS

CSS, short for Cascading Style Sheets, is a powerful language used to style the presentation of HTML documents. While HTML provides the structure and content of a web page, CSS allows you to control how that content is displayed on the screen. With CSS, you can define styles for various elements of your HTML document, such as fonts, colors, spacing, layour, and more. By separating the content from its presentation, CSS promotes cleaner code, easier maintenance, and greater flexibility in design.


Types of Cascading Style Sheets

    Inline CSS
  • Inline CSS is applied directly to HTML elements using the "style" attribute.
  • Styles defined inline override any external or internal CSS rules.

    • Internal CSS
  • Internal CSS is placed within the "style" element in the "head" section of an HTML document.
  • Styles defined internally apply to the entire document.

    • External CSS
  • External CSS is stored in separate CSS files and linked to HTML documents using the "link" element.
  • Allows for better organization and reusability of styles across multiple pages.

Introduction to GitHub

Github is a powerful platform that revolutionizes the way develops collaborate on projects, manage codebases, and streamline the software development lifecycle. Whether you're a seasoned developer or just starting out, Github offers a suite of tools and features to enhance productivity, foster collaboration, and ensure the success of your projects.

At its core, GitHub is Git repository hosting service, which means it provides a centralized location for storin and managing your Git repositories. Git, created by Linus Torvals, is a distributed version control system that allows developers to track changes to their code, collaborate with others, and maintain a historyof their project's development.

GitHub takes Git to the next level by adding a social and collaborative layer on top of the version of control functionality. Let's delve into some key features that make GitHub indispensable for developers.


Key features of GitHub:

1. Cloning Repository To Local Computer

git clone https://github.com/yourUsenameInGithub/nameOfyourRepository.git

This allows you to copy an existing repository in your account, where you can copy it anywhere on you devices.

2. Updating Local Repository

git pull

This allows you to update your current local repository which you have cloned in you computer.

3. Adding a File/Folder To Your Repository in GitHub

git add . --> for all your current local folders or git add abc.html --> specific files or git add "nameOfFOlder" --> entire folders that you can add git commit -m "YOUR DESCRIPTION TO WHAT YOU HAVE ADDED"

This allows you to add and commit your files on queue.

4. Pushing Your Queued Files Into Your GitHub Repository Online

git push

This allows you to push your files on queue in your repository folder at GitHub.

5. Amend or Rename Commit Message

git commit --amend -m "NEW DESCRIPTION TO WHAT YOU WANT TO CHANGE"

This allows you to rename your commit message.
